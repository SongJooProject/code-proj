# 주요 업무 흐름

## 뻥뚜리 문제 생성 흐름

```
1. 명세서 확인
   └── 뻥뚜리_명세서.md

2. 핵심 단어 로드
   └── law_terms_db.json

3. 원본 파일 처리
   ├── shutil.copy2()로 원본 복사
   ├── docx.Document()로 열기
   └── 문단별 빈칸 변경
       ├── 핵심 단어 검색
       ├── ▁N▁ 형식으로 교체
       └── 섹션별 번호 관리

4. 답안 생성
   ├── 【섹션 제목】 그룹핑
   └── N번. 단어 형식

5. 검증
   └── test_quiz.py 실행

6. 결과
   ├── {원본}_문제.docx
   └── {원본}_답안.docx
```

## 검증 흐름

```
1. 문제지 검증
   ├── 빈칸 형식 확인
   ├── 문단당 최대 2개
   └── 번호 유효성

2. 답안지 검증
   ├── 섹션별 번호 시작점
   ├── 번호 순서
   └── 답안 존재

3. 일관성 검증
   └── 문제지-답안 매칭
```
